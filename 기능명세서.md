# 기능 명세서

## 1. 끝말잇기

**WebSocket 연결 및 사용 방법**:

- **연결 엔드포인트**: 클라이언트는 먼저 `/ws` 엔드포인트로 WebSocket 연결을 수립해야 합니다. (SockJS fallback 지원)
- **메시지 전송**: 연결 후 클라이언트는 `/pub` prefix를 붙여서 메시지를 전송합니다. (예: `/pub/game/last-word/ready`)
- **메시지 수신**: 서버는 `/sub` prefix로 브로드캐스트하므로, 클라이언트는 해당 구독 경로를 구독해야 합니다. (예: `/sub/game/last-word/room/{roomId}`)
- **실시간 통신**: WebSocket은 양방향 실시간 통신을 지원하며, 서버에서 게임 상태 변경 시 해당 게임방의 모든 플레이어에게 자동으로 브로드캐스트됩니다.

| 구분           | 요청경로 (클라이언트 전송)   | 구독 경로 (서버 브로드캐스트)       | 기능 설명                                                                                                                                                                                                                                                                                                        | Method                   | 응답코드 |
| -------------- | ---------------------------- | ----------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------ | -------- |
| 끝말잇기 준비  | `/pub/game/last-word/ready`  | `/sub/game/last-word/room/{roomId}` | **목표**: 게임 시작 전 플레이어의 준비 상태를 업데이트하고 게임방의 모든 플레이어 상태를 조회하여 브로드캐스트<br>**주요 동작**: GameJoinVO를 받아 플레이어의 준비 상태를 업데이트하고, 게임방의 모든 플레이어 상태를 조회하여 WebSocket으로 브로드캐스트                                                        | WebSocket MessageMapping | 200 OK   |
| 끝말잇기 시작  | `/pub/game/last-word/start`  | `/sub/game/last-word/room/{roomId}` | **목표**: 끝말잇기 게임을 시작하고 첫 번째 플레이어에게 턴을 부여<br>**주요 동작**: 모든 플레이어를 준비완료 상태로 변경하고, 모든 턴을 0으로 초기화한 후 플레이어 목록의 첫 번째 플레이어(인덱스 0)의 턴을 1로 설정하여 게임 시작 상태를 브로드캐스트                                                           | WebSocket MessageMapping | 200 OK   |
| 게임 상태 조회 | `/pub/game/last-word/state`  | `/sub/game/last-word/room/{roomId}` | **목표**: 현재 게임방의 게임 상태를 조회하여 모든 플레이어에게 브로드캐스트<br>**주요 동작**: 게임방 ID를 받아 현재 게임 상태(플레이어 목록, 턴 정보 등)를 조회하고 WebSocket으로 브로드캐스트                                                                                                                   | WebSocket MessageMapping | 200 OK   |
| 단어 제출      | `/pub/game/last-word/submit` | `/sub/game/last-word/room/{roomId}` | **목표**: 플레이어가 제출한 단어를 검증하고 게임방의 모든 플레이어에게 브로드캐스트<br>**주요 동작**: Map 형태의 요청 데이터에서 userId, gameRoomId, word를 추출하여 단어의 유효성(실제 단어인지, 중복 여부, 끝말잇기 체인 유지 여부)을 검증하고, LLM을 통해 단어 설명을 생성한 후 게임 상태와 함께 브로드캐스트 | WebSocket MessageMapping | 200 OK   |

## 2. 인게임 사용자 프로필

| 구분                        | 요청경로                    | 기능 설명                                                                                                                                                                                                                                                                                                                           | Method | 응답코드 |
| --------------------------- | --------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ | -------- |
| 게임방 플레이어 조회        | `/player/{roomId}`          | **목표**: 특정 게임방에 참여한 모든 플레이어 정보를 조회<br>**주요 동작**: 게임방 ID를 받아 해당 게임방에 참여한 모든 플레이어의 정보(닉네임, 색상, 턴, 호스트 여부 등)를 조회하여 반환<br>**인증**: 인증 불필요 (public API)                                                                                                       | GET    | 200 OK   |
| 인게임 사용자 정보 업데이트 | `/player/{roomId}/{userId}` | **목표**: 게임 중 플레이어의 인게임 정보(호스트 여부, 팀 색상, 턴, 프로필 텍스트)를 업데이트<br>**주요 동작**: 경로 변수로 게임방 ID와 사용자 ID를 받고, RequestBody로 GameJoinDTO를 받아 경로 변수와 일치하는지 검증한 후 플레이어의 호스트 여부, 팀 색상, 턴 상태, 프로필 텍스트를 업데이트<br>**인증**: 인증 불필요 (public API) | PUT    | 200 OK   |

## 3. 회원가입

| 구분     | 요청경로          | 기능 설명                                                                                                                                                                                                                                        | Method | 응답코드    |
| -------- | ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------ | ----------- |
| 회원가입 | `/users/register` | **목표**: 새로운 사용자를 시스템에 등록하여 회원가입을 완료<br>**주요 동작**: 이메일 중복 검사 후, 사용자 정보를 저장. 닉네임이 없으면 이름을 닉네임으로 설정. (비밀번호 암호화는 현재 주석 처리되어 있음)<br>**인증**: 인증 불필요 (public API) | POST   | 201 CREATED |

## 4. 로그인

| 구분   | 요청경로      | 기능 설명                                                                                                                                                                                                                                                                                                          | Method | 응답코드 |
| ------ | ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------ | -------- |
| 로그인 | `/auth/login` | **목표**: 사용자 인증을 통해 JWT 액세스 토큰과 리프레시 토큰을 발급하여 로그인 처리<br>**주요 동작**: 이메일과 비밀번호를 검증하고, JWT 액세스 토큰과 리프레시 토큰을 생성. 리프레시 토큰은 HttpOnly 쿠키로 전송하고, 액세스 토큰은 응답 본문에 포함. Redis에 토큰 정보 저장<br>**인증**: 인증 불필요 (public API) | POST   | 200 OK   |

## 5. ID & PW 찾기

| 구분           | 요청경로                | 기능 설명                                                                                                                                                                                                                                                                                                                                                      | Method | 응답코드               |
| -------------- | ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ | ---------------------- |
| 이메일 찾기    | `/users/find-email`     | **목표**: 이름과 전화번호를 통해 사용자의 이메일을 찾아 반환<br>**주요 동작**: UserVO를 RequestBody로 받아 이름과 전화번호를 추출하고, 해당 정보와 일치하는 사용자의 이메일 목록을 조회하여 반환<br>**인증**: 인증 불필요 (public API)                                                                                                                         | POST   | 200 OK                 |
| 임시 토큰 발급 | `/auth/tmp-token`       | **목표**: 비밀번호 찾기 과정에서 전화번호 인증 후 임시 액세스 토큰을 발급하여 비밀번호 변경 권한 부여<br>**주요 동작**: UserVO를 RequestBody로 받아 전화번호를 추출하고, 해당 전화번호로 등록된 사용자를 찾아 임시 액세스 토큰을 발급하여 반환. 사용자를 찾을 수 없으면 404 NOT_FOUND 반환<br>**인증**: 인증 불필요 (public API)                               | POST   | 200 OK / 404 NOT_FOUND |
| 비밀번호 변경  | `/private/users/modify` | **목표**: JWT 토큰으로 인증된 사용자가 비밀번호를 포함한 회원 정보를 수정<br>**주요 동작**: Authentication 객체를 통해 인증된 사용자의 ID를 추출하고, 요청 본문의 UserVO를 받아 사용자 정보를 수정. 비밀번호가 포함된 경우 암호화하여 저장하고, 수정된 사용자 정보를 반환(비밀번호는 null로 설정하여 제외)<br>**인증**: 인증 필요 (private API, JWT 토큰 필수) | PUT    | 200 OK                 |
